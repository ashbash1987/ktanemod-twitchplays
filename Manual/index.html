Sending Messages to the Bomb

Each module will have an assigned numerical digit in order to identify between modules that may have similar appearance but different expected outcomes.
This will appear in the top-right corner of the message flyout as a command indicator of the form !x.
Start your message with that command and the remainder of the message will appear in the message flyout for that module.
Example: !1 I am a message for module 1. Hello!

You can also send messages to the bomb in general as well. To do this, start the command with the form !bomb; for example: !bomb I am a message for the bomb. Hello!

Auto-play Mode

There is also the option to enable auto-play mode, which leaves the game entirely in the hands of you and your fellow Twitch Chat experts.
In order to interact directly with the game, your commands will need to be very specific, based on what you want to interacting with.
See the details on the following pages on how to interact with the game, the bomb and each specific module.

Note: The Auto-play chat interpreter knows if a module is defused and will not interpret instructions for a defused module. Also, if given a sequence of instructions that causes a strike mid-way, it will stop the sequence mid-way to avoid further strikes.
However, it does not discern between different chat instructions, so coordinate with your fellow experts to avoid bad instructions to the bomb.

Bomb Command Reference

To pick up the bomb, use the command !bomb hold or !bomb pick up. If the bomb is already picked-up, this will do nothing.

To turn the bomb while picked up, use the command !bomb turn or !bomb turn round or !bomb turn around. If the bomb is not picked up, this will also pick the bomb up first.

To drop the bomb, use the command !bomb drop or !bomb let go or !bomb put down. If the bomb is already put down, this will do nothing.

To view the edgework of the bomb, use the command !bomb edgework. This will cycle through all the widget faces of the bomb, with short pauses on each face. If the bomb is not picked up, this will also pick the bomb up first.

Common Module Command Reference

To focus on a specific module on the bomb, use the command !x show, with 'x' being the numeric ID of the module to show.
e.g. !1 show

Some module commands below can cause extended interrupts of time. To cancel these commands and allow further commands to then be processed, use the command !cancel. This will only stop the active command, if it needs to be stopped.

If there are commands that have been issued but not yet processed, and you believe that any of the queued commands will potentially cause a strike, you can use the command !stop to stop all queued commands, as well as halting the active command like the above command.

Pre-Bomb Reference

While in the setup room before a bomb, there are various commands available:

Bomb/Mission Binder Case Commands

To pick up the binder, use the command !binder hold or !binder pick up.
To drop the binder, use the command !binder drop or !binder let go or !binder put down.
To move the active selection up and down in the binder, use the commands !binder up and !binder down respectively.
Movement commands can be chained into a list, e.g. !binder up up up down up. Also, u can be used as shorthand for up and d can be used as shorthand for down.
To choose the active selection, use the command !binder select.

Freeplay Case Commands

To pick up the freeplay case, use the command !freeplay hold or !freeplay pick up.
To drop the freeplay case, use the command !freeplay drop or !freeplay let go or !freeplay put down.
To set the number of modules, use the command !freeplay modules <x>, replacing 'x' with the requested number of modules.
To set the timer, use the command !freeplay timer <mm:ss> or !freeplay time <mm:ss>, replacing 'm:ss' with the requested time, split up in minutes ('x') and seconds ('ss').
To toggle the needy switch on and off, use the commands !freeplay needy on and !freeplay needy off respectively.
To toggle the hardcore switch on and off, use the commands !freeplay hardcore on and !freeplay hardcore off respectively.
To start a freeplay bomb, use the commands !freeplay start.

Post-Bomb Reference

At the end of a bomb there are a couple of options available:
!continue: On completing a bomb, this will take you to the main setup scene.
!back: On failing a bomb, this will take you to the main setup scene.
!retry: On failing a bomb, this will retry the bomb.

Module Command Reference (page 1)

Module Name Command & Example
Wires

Wire position is determined in order from top to bottom, first wire being 1.

To cut the wire at position 'x': cut <x>
e.g. !1 cut 5

The Button

To press and immediately release: tap or click or press and release
e.g. !1 tap or !1 click or !1 press and release

To hold the button down: hold or press
e.g. !1 hold or !1 press

To release the button with 'x' in the displayed time: release <x>
e.g. !1 release 4
Keypads

Button index is determined as the position on the module first from left to right, then top to bottom, first button being 1.
i.e. top-left is 1, top-right is 2, bottom-left is 3, bottom-right is 4.

To press a sequence of buttons: press <list>
e.g. !1 press 2 4 1 3

Simon Says

Colors can be expressed as either their English name (red, blue, green, yellow), or as their initial letter (r, g, b, y).

To press a sequence of buttons: press <list>
e.g. !1 press red red blue or !1 press r r b
Who's on First

To press a button labelled as 'x': <x>
e.g. !1 ready
Memory

To press a button in the 'x' position: position <x>
e.g. !1 position 2

To press a button labelled as 'x': label <x>
e.g. !1 label 3
Morse Code

Frequencies are expressed as the three digits after the decimal point. e.g. 3.565 Mhz would be expressed as 565.

To transmit at 'x': transmit <x>
e.g. !1 transmit 565

Module Command Reference (page 2)

Module Name Command & Example
Complicated Wires

Wire index is determined in order from left to right, first wire being 1; index does not count by the wire slots, but by the existence of wires.

To cut wires: cut <list>
e.g. !1 cut 2 4 5 6

Wire Sequences

Wires index is determined by the number slot that they are plugged in to.

To cut wires: cut <list>
e.g. !1 cut 7 8

To move down a panel: down
e.g. !1 down

To move up a panel: up
e.g. !1 up
Mazes

Maze directions can be expressed as either their English name (up, down, left, right), or as their initial letter (u, d, l, r).

To navigate the maze with a sequence of movements: move <list>
e.g. !1 move up right down down left or !1 move u r d d l
Passwords

Character column index is determined in order from left to right, first column being 1.

To cycle through the character column 'x': cycle <x>
e.g. !1 cycle 1

To try a password 'x': <x>
e.g. !1 world

Note: Submit will not be pressed unless the word can be spelled as specified.

Module Command Reference (page 3)

Module Name Command & Example
Venting Gas

To press Y: press y or press yes, or simply just y or yes
e.g. !1 press y, !1 press yes, !1 y, !1 yes

To press N: press n or press no, or simply just n or no
e.g. !1 press n, !1 press no, !1 n, !1 no

Capacitor Discharge

To hold the lever for 'x' seconds: hold <x>
e.g. !1 hold 10

Knobs

To rotate the knob 'x' number of clicks: rotate <x>
e.g. !1 rotate 3
The number that will actually be rotated by will be with respect to modulo 4.
